# Sprint C 완료 & 통합 점검 · ROI 후속 제안

## 방금 붙인 Sprint C (신뢰·전환·자신감)

- **표본 크기 배지 (`SampleSize`)**: 홈 카드·상세 실거래가 분포·비교 표·코치 견적 결과 4곳에 통일 노출. `실계약 N건 · 최근 6개월 · 신뢰 충분/보통/부족` 3단계 점 표시. → "우리는 실계약 위에서 코칭한다"는 정체성이 매 카드마다 반복된다.
- **딜러 공유용 견적 포맷**: 견적 결과에서 카톡·문자로 바로 붙여넣을 수 있는 텍스트 클립보드 복사. 차/트림/옵션/희망가/근거 한 뭉치.
- **PRO 티저 강화**: 블러 스크립트 대신 실질 가치가 보이는 두 블록으로 교체.
  - 딜러 첫 제시가 예측(오픈 수치) + 목표가 + 협상 여지
  - 저리 할부 실질금리 미니(수치 일부만 블러) — "무이자 60개월" 뒤집기 훅.

## 전체 통합 점검 (Head of Brand PD 시점)

### 잘 붙은 흐름
- 홈 → 상세 → 코치(견적 프리필) → 매장 공유의 **핵심 편도 여정**이 이제 연결됨.
- 온보딩 → 코치 프리필 → 견적 → PRO 업셀의 개인화 축이 자연스러움.
- 제보 → 리뷰 트리거로 콘텐츠 루프의 씨앗을 심음.

### 남아있는 구조적 결함 (통합 관점)
1. **홈이 여전히 "이미 유저" 전제**: 관심 차종 카드가 무조건 3장 노출됨. 워치리스트 비어있는 상태에서는 온보딩 취향 기반 추천 3장으로 대체돼야 함.
2. **관심 차종 추가 버튼이 no-op**: `<Plus/> 관심 차종 추가` 버튼이 아무 것도 안 함. Explore 또는 검색 시트로 연결 필요.
3. **탐색↔홈 정보 중복**: 홈의 "지금 뜨는 신차"와 Explore의 판매 TOP10이 같은 MOCK_CARS로 채워져 시각적으로 겹침. 홈은 "내 취향/시그널 우선", Explore는 "시장 전체"로 역할 분리 필요.
4. **비교 진입 부재**: Explore 카드/홈 트렌딩 리스트에 "+ 비교" 아이콘이 없어서, 상세 진입해야만 비교 담기가 가능. 리스트 레벨에서 한 번에 담기 필요.
5. **리뷰 데이터 소스**: 상세의 리뷰는 `REVIEWS_BY_CAR` mock이라 유저가 방금 남긴 리뷰가 반영 안 됨. 로컬 `myreviews` 병합 필요.
6. **CTA 위계 혼재**: `PrimaryButton`/`GhostButton`을 정의했지만 실제 페이지들은 아직 임시 클래스 그대로. 하단 sticky CTA·상세 CTA 사이 톤이 미묘하게 다름.
7. **인증·백엔드-UI 갭**: `watchlist`/`price_signals`/`sales_stats` 테이블 스키마는 있지만, 홈·상세는 여전히 `MOCK_CARS`만 읽음. 실 데이터 연결이 벌어져 있음.
8. **마이 페이지 빈약**: 저장한 견적·비교함·내 리뷰·제보 이력이 마이 탭에 모여있지 않음. 재방문 훅 부족.
9. **오류/빈 상태 카피 부재**: 비교 <2대·워치리스트 0건·리뷰 0건 상태의 안내가 밋밋함.
10. **접근 경로가 없는 화면**: `/diagnose`는 홈 하단 카드 하나로만 접근 가능. 상세의 "이 견적서 함정 체크" 진입점이 없어 사용률이 뜰 수가 없음.

## ROI 순 후속 작업 제안

### 티어 1 — 노력 대비 임팩트 최상 (다음 스프린트 후보)
1. **홈 빈-상태 & 워치리스트 실 데이터화** *(사용성↑, 재방문 훅)*
   워치리스트가 비면 온보딩 취향으로 3대 추천, 있으면 실 데이터. `watchlist` 테이블 read/write 연결. "관심 차종 추가"가 진짜 동작.
2. **상세 → 진단 진입점** *(전환↑)*
   상세의 스티키 CTA 옆 "이 차 견적 함정 체크" 세컨더리 링크. 견적을 이미 받은 사람의 다음 액션 자연 연결.
3. **리스트 레벨 "비교 담기"** *(비교 사용률↑)*
   홈 트렌딩·Explore 카드에 작은 + 아이콘. 상세 안 들어가도 2~3대 담기 가능.
4. **마이 페이지 허브화** *(리텐션)*
   저장 견적/비교함/내 리뷰/제보 상태를 카드 4개로 노출. 각각 딥링크.

### 티어 2 — 방향성·정체성 강화
5. **홈 vs Explore 역할 분리** *(정보구조)*
   홈: 내 시그널 위주(관심·오늘의 시그널·개인화 추천). Explore: 시장 지도·랭킹만. 중복 리스트 제거.
6. **리뷰 로컬 병합 + 정렬 옵션** *(콘텐츠 루프)*
   `getMyReviews()`를 상세 리뷰 상단에 "내가 남긴 리뷰"로 표시. 실오너 배지·최신순 정렬.
7. **CTA·버튼 통일** *(브랜드 정합성)*
   전 페이지 sticky/primary/ghost 버튼을 `PrimaryButton`/`GhostButton`으로 리팩터. 톤 통일.

### 티어 3 — 백엔드/성장 인프라 (비용 큼)
8. **실 데이터 서버 함수화**: `MOCK_CARS`를 `vehicles + trims + price_signals + sales_stats` join 쿼리로 대체. `createServerFn` + 캐시.
9. **PRO 결제 붙이기**: Stripe/Toss. 실질금리 계산기·협상 스크립트 진짜 언락.
10. **제보 검증 파이프라인**: 계약서 OCR + 관리자 검수 큐. 정확도가 우리 해자.
11. **알림(가격/시그널 변화)**: 워치리스트 차종의 시그널이 바뀌면 푸시/이메일. 재방문 트리거.
